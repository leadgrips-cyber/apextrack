Publisher API

Read your offers, tracking links, and performance reports programmatically — the same data your dashboard shows, available to scripts, cron jobs, and external tracking platforms.

All endpoints are read access plus tracking-link creation. Nothing here can touch another publisher's data, and nothing here can approve, edit, or delete offers — that stays admin-only.

Authentication

Generate a key from Dashboard → API Access. Every request needs it in one of these forms:

HeaderExample
AuthorizationBearer pub_5f1e2a...
X-API-Keypub_5f1e2a...

The key is shown once, in full, right after you generate or regenerate it — only its hash is stored, so save it somewhere safe (a secrets manager or environment variable). Regenerating immediately revokes the previous key.

Quickstart

  1. Log into your ApexTrack dashboard.
  2. Open API Access in the sidebar.
  3. Click Generate Token (or Regenerate if you already have one) and copy the key immediately.
  4. Call GET /api/v1/me to confirm it works (see examples below).
  5. Pull your offer catalog with GET /api/v1/offers, generate a tracking link with POST /api/v1/tracking-links, and start reading reports.

Base URL

https://<your-tracking-domain>/api/v1

Use the tracking domain shown in your dashboard. Locally during development that's typically http://localhost:3000/api/v1.

Your first call

curl "https://<your-tracking-domain>/api/v1/me" \
  -H "Authorization: Bearer pub_5f1e2a..."
{
  "publisher_id": "09f109dc-55e0-4c8d-9722-258d96f6377a",
  "email": "you@example.com",
  "scopes": ["read", "write"]
}

Error codes

StatusCodeMeaning
401MISSING_API_KEYNo key was found on the request.
401INVALID_API_KEYThe key doesn't match any account.
401REVOKED_API_KEYA newer key has replaced this one.
401EXPIRED_API_KEYThe key has passed its expiration date.
403ACCOUNT_NOT_ACTIVEYour publisher account isn't active.
403OFFER_NOT_APPROVEDYou haven't been approved for this offer yet.
404OFFER_NOT_FOUNDNo live offer with that ID (also returned for offers that exist but aren't active — details are never revealed for those).
404TRACKING_LINK_NOT_FOUNDNo such link, or it belongs to a different account.
400VALIDATION_ERRORA required field is missing or malformed.

Every error response is JSON: {"error": "CODE", "message": "human-readable explanation"}.

Offers

EndpointDescription
GET /offersEvery live (ACTIVE) offer in the catalog, with an is_approved flag for your account.
GET /offers/approvedOnly offers you're approved to run.
GET /offers/:idFull detail for one offer, including its available conversion events.
GET /offers/:id/creativesCreative assets. Requires approval if the offer is approval-gated.
GET /offers/:id/landing-pagesActive landing pages. Same approval rule as creatives.
GET /offers/:id/targetingAllowed geos/devices and any additional targeting rules.
GET /categoriesAll active offer categories.

Offers that are DRAFT, PAUSED, ARCHIVED, CLOSED, or EXHAUSTED never appear and always 404 by ID — there is no way to discover an offer that isn't currently live.

Tracking links

EndpointDescription
GET /tracking-linksEvery tracking link you've generated.
POST /tracking-linksGenerate one for an offer (creates it if it doesn't exist yet, otherwise returns the existing one for that offer+sub combination).
GET /tracking-links/:idLook up one of your links by ID.

Example: generate a tracking link

POST /api/v1/tracking-links
Content-Type: application/json

{ "offer_id": 9, "sub1": "my_campaign_42" }
{
  "tracking_link": {
    "id": "a877b0bf-898f-47b0-b59d-07699f44c781",
    "offer_id": 9,
    "sub1": "my_campaign_42",
    "tracking_url": "/api/click?offer_id=9&publisher_id=...&sub1=my_campaign_42"
  }
}

Prefix tracking_url with your tracking domain to get the full clickable link.

Reports

EndpointDescription
GET /reports/clicksPaginated click log.
GET /reports/conversionsPaginated conversion log, including pending/review-queue ones.
GET /reports/dailyClicks, conversions, payout, revenue, EPC, and CR grouped by day.
GET /reports/eventsConversion counts and payout broken down by event (Lead, Sale, Deposit, …) per offer.
GET /reports/summaryTotals: clicks, conversions, payout, revenue, EPC, CR, RPC, and your current wallet balances.

Query parameters (all report endpoints)

ParamApplies toNotes
from, toallISO date/time bounds, e.g. 2026-07-01.
offer_idclicks, conversionsFilter to one offer.
statusconversionse.g. APPROVED, REVIEW_QUEUE, PENDING, PAID, REJECTED.
page, page_sizeclicks, conversionsDefault page size 50, max 200.

Definitions

EPC = payout ÷ clicks (what you earn per click). CR = approved conversions ÷ clicks × 100. RPC = revenue ÷ clicks. Payout and revenue totals only ever include settled conversions (APPROVED/PAID); the conversion count itself includes everything except conversions explicitly marked REJECTED, so pending and review-queue conversions are always visible in your reports.

cURL examples

# List approved offers
curl "https://<your-domain>/api/v1/offers/approved" \
  -H "Authorization: Bearer pub_5f1e2a..."

# Generate a tracking link
curl -X POST "https://<your-domain>/api/v1/tracking-links" \
  -H "Authorization: Bearer pub_5f1e2a..." \
  -H "Content-Type: application/json" \
  -d '{"offer_id": 9, "sub1": "campaign_42"}'

# Conversion report for a date range, approved only
curl "https://<your-domain>/api/v1/reports/conversions?from=2026-07-01&to=2026-07-31&status=APPROVED" \
  -H "Authorization: Bearer pub_5f1e2a..."

JavaScript (browser/fetch)

const API_KEY = "pub_5f1e2a...";
const BASE = "https://<your-domain>/api/v1";

async function apextrack(path, options = {}) {
  const res = await fetch(`${BASE}${path}`, {
    ...options,
    headers: {
      "Authorization": `Bearer ${API_KEY}`,
      "Content-Type": "application/json",
      ...(options.headers || {}),
    },
  });
  if (!res.ok) {
    const body = await res.json().catch(() => ({}));
    throw new Error(`${res.status} ${body.error || ""}: ${body.message || res.statusText}`);
  }
  return res.json();
}

const { offers } = await apextrack("/offers/approved");
const { tracking_link } = await apextrack("/tracking-links", {
  method: "POST",
  body: JSON.stringify({ offer_id: offers[0].id, sub1: "js_demo" }),
});
console.log(tracking_link.tracking_url);

Node.js

// npm install node-fetch   (or use the global fetch on Node 18+)
const API_KEY = process.env.APEXTRACK_API_KEY;
const BASE = "https://<your-domain>/api/v1";

async function getSummary() {
  const res = await fetch(`${BASE}/reports/summary`, {
    headers: { "Authorization": `Bearer ${API_KEY}` },
  });
  if (!res.ok) throw new Error(`ApexTrack API error: ${res.status}`);
  return res.json();
}

getSummary().then((summary) => {
  console.log(`Payout: $${summary.total_payout} | CR: ${summary.cr}% | EPC: $${summary.epc}`);
});

PHP

<?php
$apiKey = getenv('APEXTRACK_API_KEY');
$base = 'https://<your-domain>/api/v1';

function apextrack_get(string $path, string $apiKey, string $base): array {
    $ch = curl_init($base . $path);
    curl_setopt_array($ch, [
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_HTTPHEADER => ["Authorization: Bearer $apiKey"],
    ]);
    $response = curl_exec($ch);
    $status = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);

    $data = json_decode($response, true);
    if ($status >= 400) {
        throw new Exception("ApexTrack API error {$status}: " . ($data['message'] ?? 'unknown'));
    }
    return $data;
}

$offers = apextrack_get('/offers/approved', $apiKey, $base);
foreach ($offers['offers'] as $offer) {
    echo $offer['name'] . " — payout \${$offer['payout_amount']}\n";
}

Connecting an external tracker

To wire ApexTrack into Affise, Trackier, Binom, Voluum, RedTrack, BeMob, Thrive, or a custom in-house tracker:

  1. Import the offer catalog — poll GET /offers/approved on a schedule (hourly is typical) and sync name, payout, geos/devices, and landing page URL into your tracker's offer list.
  2. Generate one tracking link per offer via POST /tracking-links, or build your own click URL directly: {tracking_domain}/api/click?offer_id={id}&publisher_id={your_publisher_id}&sub1={your_click_id} — pass your external tracker's click ID as sub1 so you can reconcile later.
  3. Pull performance back into your tracker — poll GET /reports/conversions (filtered with from/to) on a schedule and match rows back to your clicks using the sub1 value stored on each click.
  4. Reconcile revenue — use GET /reports/summary or /reports/daily for the numbers your finance/reporting stack needs (payout, EPC, CR).
Note on conversion postbacks: ApexTrack's server-to-server conversion endpoint (/api/postback) is a separate, offer-level integration your account manager configures per advertiser — it is not part of this key-authenticated reporting API. Talk to your account manager about postback setup for a specific offer.
